# litools

litools 是一个基于 Rust / Tauri / SolidJS 的桌面效率工具平台，目标是做成 uTools 风格的本地启动器，并逐步扩展为插件驱动的本地工具生态。

## 当前已完成基础

- [x] Rust workspace foundation
- [x] Tauri desktop shell
- [x] Built-in command search provider
- [x] SQLite schema foundation
- [x] Plugin manifest and permission architecture skeleton

## Roadmap

### Phase 1：本地启动器 MVP

目标：先做出一个可以日常使用的 uTools 风格本地启动器。这个阶段的重点不是插件生态，而是唤起、输入、搜索、执行这些基础体验必须稳定。

#### 1.1 面板交互稳定

优先级最高。Tab / 键盘选择 / Enter 执行 / Escape 行为必须先稳定。

- [x] 输入框永远是主焦点
- [x] ArrowUp / ArrowDown / Tab / Shift+Tab 行为明确
- [x] Enter 执行当前选中项
- [x] Escape 清空或隐藏窗口
- [x] 鼠标 hover 只改变视觉，不改变键盘选中索引
- [x] 鼠标点击可直接执行
- [x] 结果列表滚动时，键盘选中项自动进入可视区域
- [x] 空结果 / loading / error 状态稳定

#### 1.2 原生窗口体验稳定

目标：更像原生启动器，而不是网页弹窗。

- [x] 无系统边框
- [x] 透明窗口背景
- [x] 圆角无白底 / 黑边 / 阴影异常
- [x] 输入区可拖动窗口
- [x] 失焦隐藏
- [x] 快捷键显示 / 隐藏稳定
- [ ] 多屏幕下显示位置合理
- [x] 显示时自动聚焦输入框
- [ ] macOS 下窗口层级、激活状态正确

#### 1.3 系统应用搜索 MVP

系统应用搜索是 uTools 风格启动器的核心能力，属于 Phase 1 验收项，而不是后续增强项。

- [x] 索引 `/Applications`
- [x] 索引 `~/Applications`
- [x] 解析 `.app` 名称
- [x] 获取 bundle id
- [x] 获取应用 icon
- [x] 支持打开应用
- [x] 最近打开应用优先

#### 1.4 内置命令补齐

内置命令用于保证启动器自身功能闭环。

- [x] 打开设置
- [x] 打开诊断
- [x] 打开插件管理
- [x] 切换主题
- [x] 重建索引
- [x] 隐藏窗口
- [x] 退出应用
- [x] 打开日志目录
- [x] 打开数据目录

#### 1.5 基础结果体验

- [x] 应用结果展示真实图标
- [x] 命令结果展示统一图标
- [x] 结果卡片支持副标题或来源，但保持紧凑
- [x] 常用项排序靠前
- [x] 最近使用结果优先
- [x] 搜索关键词高亮
- [x] 支持基础模糊匹配

#### Phase 1 验收标准

- [x] 全键盘可操作
- [x] 快捷键唤起后输入框稳定聚焦
- [x] 可以搜索并打开系统应用
- [x] 可以执行内置命令
- [x] 鼠标和键盘状态不互相打架
- [x] loading / empty / error 状态稳定
- [ ] 通过前端格式化和校验

### Phase 2：插件系统 MVP

目标：让第三方能力可以被接入，但先保持克制。第一版只解决本地插件发现、加载、注册命令和管理，不急着做 marketplace、复杂沙箱或完整插件页面。

#### 2.1 插件 Manifest

- [x] Manifest 和权限架构骨架
- [ ] 插件 ID
- [ ] 名称
- [ ] 版本
- [ ] 作者
- [ ] 入口文件
- [ ] 权限声明
- [ ] 贡献的命令
- [ ] 配置项 schema

Manifest 示例：

```json
{
  "id": "example.plugin",
  "name": "Example Plugin",
  "version": "0.1.0",
  "commands": [],
  "permissions": []
}
```
